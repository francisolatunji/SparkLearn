# SparkLearn - Product, Animation, and Learning System (Duolingo-Level)

## 1. The Real Product (Not What You Think)

You are not building:

> "an app that teaches electronics"

You are building:

> **a habit-forming system that delivers tiny wins through electronics**

Everything below serves that.

## 2. Core Loop (This Is Everything)

Every top app has a loop.

`Tap -> Learn -> Answer -> Feedback -> Reward -> Repeat`

If this loop is:

- fast -> users stay
- slow -> users leave

Target:

> **5-15 seconds per interaction**

## 3. Learning Design (This Is Where You Beat Others)

Most apps dump information.

You will structure learning like this:

### 3.1 Micro-Lesson Formula

Every lesson:

`Concept (5 sec) -> Question (10 sec) -> Feedback (5 sec) -> Reward (2 sec)`

Total:

> ~20 seconds

### 3.2 Concept Compression

Bad:

> "A resistor is a component that limits current..."

Good:

> "Resistor = slows electricity"

You expand later. Not at the start.

### 3.3 Learning Sequence (Critical)

You MUST order concepts like this:

#### Level 1 - Intuition

- Electricity flows like water
- More voltage = stronger push

#### Level 2 - Control

- Resistors slow flow
- LEDs light up

#### Level 3 - Logic

- Circuits work or fail
- Fixing mistakes

#### Level 4 - Systems

- Arduino
- Sensors
- Real builds

## 4. Interaction Design (Senior-Level Detail)

Every tap must:

- Change state
- Give feedback
- Move progress

If not -> it's dead UI

### 4.1 Question Interaction States

`Idle -> Hover -> Selected -> Locked -> Feedback -> Transition`

### 4.2 Timing (Extremely Important)

- Tap response: **<100ms**
- Feedback appears: **instant**
- Transition: **300-500ms**

Slow = boring  
Fast = addictive

## 5. Animation System (Duolingo-Level Thinking)

Animation is not decoration.  
It is **communication + reward**.

### 5.1 Animation Hierarchy

#### Level 1 - Functional

- Button press
- Screen transitions

#### Level 2 - Feedback

- Correct / incorrect
- XP gain

#### Level 3 - Emotional

- Mascot reactions
- Celebrations

### 5.2 Motion Principles

- Ease-out (natural feel)
- Slight bounce (playful)
- Never linear motion

### 5.3 Reward Density

Every:

- correct answer -> reward
- streak -> reward
- completion -> reward

No dead moments.

## 6. Mascot System (Senior Design)

Your mascot is not decoration.

It is:

> **a real-time emotional interface**

### 6.1 Behavior Rules

- Reacts within 300ms
- Mirrors user success/failure
- Occasionally initiates interaction

### 6.2 Dialogue Style

Bad:

> "Great job!"

Good:

> "You're getting fast."

Bad:

> "Incorrect"

Good:

> "Close. Think about current flow."

### 6.3 Presence

- Always visible in lessons
- Not blocking content
- Slight idle motion

## 7. What Duolingo Gets WRONG (Your Opportunity)

This is where you win.

### 7.1 Lack of Real Understanding

Duolingo = memorization  
You = **mental models**

Your upgrade:

- show current flowing
- show LED lighting
- show wrong wiring failing

### 7.2 Weak Depth Transition

Duolingo struggles going beginner -> advanced

Your upgrade:

- Introduce "Why it works"
- Introduce "What happens if..."

### 7.3 No Real-World Bridge

You fix this with:

- Arduino examples
- Real circuits
- Build challenges

## 8. Advanced Feature (Senior-Level Differentiation)

### 8.1 Circuit Simulation Lite

Not full simulator. Just:

- show flow
- show outcome

### 8.2 Error-Based Learning

Instead of:

> "What is correct?"

Ask:

> "Why is this broken?"

This is **10x more powerful**.

### 8.3 Progressive Difficulty

Track:

- accuracy
- speed

Adjust:

- question difficulty
- repetition

## 9. Engagement System (Retention Engine)

### 9.1 Daily Loop

`Open app -> Continue -> 3-5 lessons -> Reward -> Exit`

Time:

> 2-5 minutes total

### 9.2 Comeback Hooks

If user leaves:

- Mascot message
- Streak warning
- Quick restart

### 9.3 Session Design

Every session must:

- start instantly
- end with reward

## 10. UI System (Senior Simplicity)

If you need to explain your UI:

> it's too complicated

Rules:

- 1 main action per screen
- 3 max choices
- No scrolling during lessons
